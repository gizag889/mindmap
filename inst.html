📱 AIマインドマップ自動生成モバイルアプリ 開発指示書

1. プロジェクトの目的とコアバリュー

ユーザーが入力したテキスト（チャット）から、AIが自動的にマインドマップを生成・拡張していく「モバイル特化型のアイデア出し（ブレインストーミング）アプリ」を開発する。
直感的な操作性と、AIによる「共感・拡張型」のサポートにより、ユーザーの思考を止めずにアイデアを広げる体験を提供する。

2. 指定技術スタック

以下の技術構成を前提としてコードを生成してください。

  • フロントエンド: React Native (Expo)
  • 描画・ジェスチャー: react-native-svg, react-native-gesture-handler
  • アニメーション: react-native-reanimated (滑らかなトランジション必須)
  • UIコンポーネント: @gorhom/bottom-sheet (チャットUI用)
  • バックエンド・API中継: Cloudflare Workers (LLM APIへのセキュアなプロキシ)
  • データベース: Cloudflare D1 (チャット履歴とマインドマップ構造の保存)

3. 画面遷移とUI/UXの仕様

フロー1：初期起動時のマジックトランジション

  1. 初期画面: 通常のチャットUIのみを表示。「整理したいテーマを教えてください」とAIが尋ねる。
  2. 初回入力: ユーザーが最初のテキストを送信する。
  3. 遷移アニメーション: ローディング後、チャット画面がフェードアウト（または下へ沈み込む）。
  4. マップ展開: 画面中央に最初の「ルートノード（中心テーマ）」が出現し、そこから自動生成された複数の「子ノード」がアニメーションと共に枝分かれして広がる。

フロー2：マインドマップの表示モード切り替え

画面上部のトグル等で以下の2モードを切り替え可能にする。

  • 全体表示（俯瞰モード）: ピンチイン・アウト、スワイプでのパン操作が可能。すべてのノードとエッジを表示し、全体の広がりを把握できる。
  • アクティブ表示（フォーカスモード）: 選択中（Active）のノードを中心に自動スクロール・ズームする。選択中ノード、その親ノード、直接の子ノードのみを強調表示し、関係のない枝は透過（opacity低下）させる。

フロー3：ノードタップ時のボトムシート展開

  1. ユーザーがマップ上のノードをタップする。
  2. 画面下部から @gorhom/bottom-sheet を用いたチャットUIがスッと立ち上がる。画面上半分にはタップしたノードがフォーカス表示されている状態を保つ。
  3. ボトムシート内でそのノード（文脈）に関するチャットを行う。
  4. AIの回答完了に合わせて、フォーカスされているノードから新たな子ノードが自動的に描画・追加される。

4. データ構造 (JSON仕様)

バックエンドのLLM（Cloudflare Workers経由）は、テキストの回答と同時に、マップ描画用の構造化データを以下のJSON形式で返すようにシステムプロンプトを構成すること。

{
  "chatResponse": "ユーザーへの自然な共感とアイデアの返答テキスト",
  "mindMapUpdates": {
    "nodes": [
      {"id": "node_1", "label": "中心テーマ", "parentId": null},
      {"id": "node_2", "label": "派生アイデアA", "parentId": "node_1"}
    ],
    "edges": [
      {"source": "node_1", "target": "node_2"}
    ]
  }
}

5. AIアシスタントのキャラクター（システムプロンプト要件）

LLMに対するシステムプロンプトには、以下の指示を厳格に含めること。

  • 役割: 共感・拡張型のアシスタント。ユーザーのアイデアを絶対に否定せず、肯定から入ること。
  • ノード生成のルール: ユーザーの抽象的な入力（例：「もっとキラキラした感じ」）を、マップに適した簡潔で構造的な単語（例：「視覚効果：ラグジュアリー」）に変換してノード化すること。
  • 拡張性: 1つの入力に対して、可能な限り自動で「3つの異なる切り口（枝）」を提案し、出力JSONに含めること。

6. 実装の優先ステップ（Antigravityへの指示）

まずは以下のStep 1から実装を開始し、動作確認ができるコードを出力してください。

  1. Step 1: Expo環境における、react-native-svgとreact-native-reanimatedを用いた「ツリー構造データからのマインドマップ描画」と「ズーム/パン操作」の基礎コンポーネント作成。
  2. Step 2: @gorhom/bottom-sheetの導入と、ノードタップ時の連動アニメーション（フォーカス＆シート展開）の実装。
  3. Step 3: 初期画面（チャットUI）からマインドマップ画面へのトランジション実装。
  4. Step 4: Cloudflare Workersを用いた、フロントエンドからのメッセージ受け取りとダミーJSONデータを返すAPIエンドポイントの構築。
